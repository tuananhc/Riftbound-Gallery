# Thursday 17 April 2026 - Dev Log No. 1

Built a full deck builder feature into the card gallery — a guided two-step flow (Legend → Champion → free build) with a fixed slide-in panel, per-type copy limits, card indicators, and a glowing card counter. The work touched four components and introduced a new extracted component, `DeckPanel.js`.

---

## Decisions Made

- **Prop drilling over React Context for deck state**
  Deck state (`deck`, `legendId`, `championId`) lives in `CardGrid.js` and is passed down to `DeckPanel`, `CardModal`, and `CardItem` as props. Context was considered but rejected — the component tree is only three levels deep, and explicit props keep data flow visible without forcing all consumers to re-render on every deck change.

- **`legendId` / `championId` as slot pins, not positional array indices**
  The legend and champion slots are tracked as IDs into the `deck` array rather than `deck[0]`/`deck[1]`. This makes the slot assignments resilient to reordering and removal, and avoids off-by-one errors when a card is decremented out of the deck.

- **Legend excluded from deck sections; Champion included in Units**
  The Legend card appears only in the top `ChosenChampionSection` and is excluded from all regular `SECTIONS` via `card.id !== legendId`. The Chosen Champion is shown in both the top section (display-only) and in the Units section (with controls), because the top slot is decorative and the section controls are where copy management happens.

- **`maxCopies` as an inline helper, not a validator module**
  The copy limit logic (`Legend → 1, all others → 3`) is simple enough to live as a one-line helper in `CardGrid.js`. A full `src/lib/deck/validator.js` was discussed as the right location if validation logic grows (export rules, required Legend + Champion check), but was not implemented this session.

- **`overflow: "visible"` on `CardItem` to allow badge overflow**
  The deck count badge is positioned at `bottom: -14px` to centre on the card's bottom border. The card div originally had `overflow: "hidden"` which clipped it. Changed to `overflow: "visible"` as the simplest fix — the card image uses an inner `<Image>` that handles its own bounds.

## Topics Discussed

- **Slide-in drawer vs. separate screen** — discussed UX trade-offs; chose slide-in panel so the card gallery remains visible while building, allowing immediate browsing and adding.
- **Where to put deck validation logic** — explored `CardGrid.js` vs. a dedicated `src/lib/deck/validator.js`. Agreed that `validator.js` is the right long-term home (pure function, testable, reusable for export validation), but deferred implementation.
- **Material UI substitution audit** — reviewed which MUI components could be replaced with current inline-style approach. Concluded the project's design is custom enough that MUI would add bundle weight without UI benefit; the existing approach covers all current needs.
- **CSS consolidation** — briefly discussed moving repeated `font-family` declarations to `global.css`; deprioritised in favour of completing deck builder features.
- **Guided build flow design** — explored enforcing Legend + Champion selection before free building. Settled on auto-filter transitions (Legend added → filter to Champion; Champion added → clear filters) rather than a blocking modal or wizard, keeping the experience fluid.
- **Context vs. prop drilling** — walked through the trade-off for passing deck state to `CardModal`. Kept prop drilling because the tree depth doesn't justify a provider.

## Bug Fixes

- **`setDeckMode` stale reference in JSX**
  After `deckMode` state was renamed to `deckOpen`, a reference to `setDeckMode` remained in the JSX. Fixed by finding and replacing all occurrences with `setDeckOpen`.

- **`deck.slice(0, 2)` positional assumption for champion section**
  An earlier attempt used `deck[0]` and `deck[1]` to derive the legend and champion entries. This broke when cards were removed or added in non-legend-first order. Replaced with `deck.find(({ card }) => card.id === legendId)` / `deck.find(({ card }) => card.id === championId)`.

- **Count badge clipped by `overflow: "hidden"`**
  The badge at `bottom: -14px` was invisible because `CardItem`'s wrapper div had `overflow: "hidden"`. Fixed by changing to `overflow: "visible"`.

- **MD036 linting warnings in dev-log markdown**
  Bold text used as section headings (`**Top — ...**`) triggered MD036 warnings. Fixed by replacing with `####` ATX headings.
