## Architecture Decisions

### Filtering Strategy

**Decision:** Fetch all cards client-side, filter in memory via `useMemo`.

**Why:** 950 cards at ~50KB gzipped is negligible. Client-side filtering is instant with no network latency per interaction. Server-side filtering would add 100‚Äì400ms per filter change plus debouncing, loading states, and error handling complexity for a worse user experience.

---

### Database Migration ‚Äî Neon ‚Üí DynamoDB

**Decision:** Drop Neon Postgres entirely, move cards to DynamoDB alongside users, decks, and match history.

**Why:** The original justification for Neon was server-side SQL filtering. Once filtering moved client-side, SQL offered no advantage. DynamoDB is AWS-native, has a permanent free tier, and eliminates a second database dependency. A simple full table Scan handles 950 cards cleanly.

---

### DynamoDB Configuration

**Billing:** On-demand across all tables.

**Table class:** Standard ‚Äî cards are read on every page load making frequent-access Standard cheaper than infrequent-access Standard-IA.

**Tables finalised:**

- Cards:
  - Partition Key: `id` - Card Number
  - Sort Key: `name`
  - Notes: Full Scan on load

---

### Deck Design

**Decision:** Separate Decks and DeckVersions tables rather than embedding decks in Users.

**Why:** Card + count combinations and full version history per deck would grow too large for DynamoDB's 400KB item limit if embedded. Version control requires snapshots on every save. Restoring a version creates a new version rather than overwriting history, preserving a full audit trail. Saves use `TransactWriteCommand` to atomically write both the Decks update and DeckVersions snapshot.

## Server / Client Architecture Migration

Migrated `page.js` from a monolithic client component to a proper server/client split:

- `page.js` ‚Äî removed `"use client"`, now a server component that calls `getAllCards()` and passes results down as props
- `CardGrid.js` ‚Äî `"use client"`, owns all filtering, sorting, and search state via `useCardFilters` hook
- `CardItem.js` ‚Äî extracted into its own file, displays card image only
- `CardFilters.js` ‚Äî sidebar filters extracted into its own file
- `CardModal.js` ‚Äî modal extracted into its own file

**Why:** `postgres` is a Node.js package that cannot run in the browser. The DB fetch must live in a server component. Client components receive data via props and handle interactivity only.

## UI Changes

### Card Text Parser

Created `src/lib/cardTextParser.js` ‚Äî detects `[Keyword]` patterns in card ability and effect text and maps them to styles via a `KEYWORD_STYLES` dictionary. Unknown keywords fall back to `DEFAULT_KEYWORD_STYLE`. Created `CardText.js` component to render parsed segments with parallelogram-shaped keyword badges using `skewX(-12deg)`.

### Next.js Image Config

Added `cmsassets.rgpub.io` and `assetcdn.rgpub.io` to `remotePatterns` in `next.config.js` to allow `<Image>` to load card images from external CDN domains.

## Bug Fixes

**Server/client boundary errors** ‚Äî Three pages had `onMouseEnter`/`onMouseLeave` event handlers on server components, which Next.js does not allow. Fixed by replacing with CSS hover classes (`.featured-card`, `.back-link`).

**Hydration mismatch** ‚Äî Inline `<style>` tags were causing SSR/client mismatches. Resolved by adding `suppressHydrationWarning`.

**`Module not found: fs`** ‚Äî `getAllCards` (a Node.js DB import) was imported inside a file marked `"use client"`. Fixed by removing the stray import and ensuring all DB calls live in server components only.

**`undefined.md` on Vercel** ‚Äî `[slug]/page.js` was accessing `params.slug` directly instead of `await props.params`, which is required in Next.js 15. Fix was applied locally and resolved after redeploy.

**`.next` folder tracked by git** ‚Äî Was in `.gitignore` but had been committed before the rule was added. Fixed with `git rm -r --cached .next`.

**Vercel output directory** ‚Äî Vercel was configured to look for a `build` folder. Fixed by setting the output directory to `.next` in Vercel project settings.
