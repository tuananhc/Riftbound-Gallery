# Riftbound — Dev Log

**Date:** Saturday 18 April 2026

## Deck Builder

Built a full deck builder feature into the card gallery. The panel slides in from the right side of the screen when the **Deck Builder** button is clicked next to the sort controls. Opening the panel activates deck mode — cards show a `+` overlay on hover and a lime border + count badge once added.

### Build Flow

Deck building is guided in two mandatory steps before free building begins:

1. **Legend selection** — opening deck builder auto-filters to Legend cards. Adding a Legend locks it as the deck's Legend slot, then automatically switches the filter to Champion and prefills the search with the Legend's name to surface matching Champions.
2. **Champion selection** — adding a Champion locks it as the Chosen Champion slot and clears all filters for free building.

Removing the Legend resets both slots and returns the filter to Legend, restarting the flow.

### Copy Limit Rules

- Legend cards: max 1 copy
- All other cards: max 3 copies

The `maxCopies(card)` helper in `CardGrid.js` enforces this at the `addToDeck` level. The card modal's **ADD TO DECK** button also respects the limit and shows `(1/1)`, `(2/3)` etc. as copies are added, switching to **MAX COPIES** when the cap is reached.

### Panel Layout

The deck panel is split into two zones:

#### Top — Legend & Chosen Champion

Displayed as rich cards showing the card image thumbnail, name, and domain color icons. The Legend has a `−` remove button. The Chosen Champion is display-only — no controls — since its copies are managed in the sections below.

#### Bottom — Deck Sections

Cards are grouped into labelled sections: **Battlefields**, **Runes**, **Units** (includes Champions), **Spells**, **Gears**. Sections only render if they contain cards. Each row has `−` / count / `+` controls. The Legend card is excluded from all sections; the Chosen Champion appears in Units with normal copy controls.

### Card Counter

The `X/40 CARDS` counter in the panel header glows red while building and switches to green when the deck hits exactly 40 — with a background tint, outer box-shadow, and text-shadow all transitioning together.

### Components

- `CardGrid.js` — owns all deck state: `deck`, `legendId`, `championId`, filter transitions on add/remove
- `DeckPanel.js` — slide-in panel, receives state and callbacks as props
- `CardModal.js` — shows **ADD TO DECK** button when deck builder is open

## Card Item — Deck Mode Indicators

In deck builder mode, cards in the gallery show:

- **Lime green outline** (`outline: 2px solid #34ed40`) when the card is in the deck
- **Count badge** centered on the bottom border when count > 0

The badge sits outside the card's `overflow: hidden` clip by using `bottom: -14px` on an absolutely positioned element.

---

## Architecture Notes

### Props over Context

Considered using React Context to pass deck state down from `CardGrid` to `CardModal`. Kept prop drilling instead — the component tree is shallow (three levels), and explicit props make data flow visible without introducing a provider that would force all consumers to re-render on every deck change.
